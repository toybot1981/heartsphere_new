# "我的家庭"场景初始化说明

本脚本用于在配置管理中预置"我的家庭"场景及其8个角色。

## 使用方式

### 方式一：直接执行SQL脚本（推荐）

1. **执行SQL脚本**：
   ```bash
   mysql -u root -p heartsphere < init_family_scene_data.sql
   ```

2. **在资源管理中替换图片**：
   - 登录管理后台
   - 进入"资源管理"页面
   - 找到已创建的资源（如"爸爸-头像"、"爸爸-背景"等）
   - 点击编辑，上传真实的头像和背景图片
   - 更新角色的头像和背景URL

### 方式二：通过管理后台手动创建（推荐用于首次使用）

#### 第一步：在资源管理中添加图片

1. 登录管理后台
2. 进入"资源管理"页面
3. 选择分类为 `character`
4. 依次上传以下资源的图片：

**头像资源（分类：character）**：
- 爸爸-头像
- 妈妈-头像
- 女儿-头像
- 儿子-头像
- 爷爷-头像
- 奶奶-头像
- 姥姥-头像
- 姥爷-头像

**背景资源（分类：character）**：
- 爸爸-背景
- 妈妈-背景
- 女儿-背景
- 儿子-背景
- 爷爷-背景
- 奶奶-背景
- 姥姥-背景
- 姥爷-背景

#### 第二步：创建"我的家庭"场景

1. 进入"场景管理"页面
2. 点击"新增场景"
3. 填写以下信息：
   - **名称**：我的家庭
   - **描述**：一个温馨的家庭场景，包含家庭成员之间的日常互动和情感交流。
   - **场景图片**：选择一个温馨的家庭场景图片（可在资源管理中选择 `era` 分类的资源）
   - **是否启用**：是
   - **排序**：100
4. 点击"保存"

## 注意事项

1. **图片资源**：SQL脚本中使用了占位图片URL（picsum.photos），实际使用时应在资源管理中上传真实的头像和背景图片，然后更新角色的图片URL。

2. **场景关联**：创建角色时，务必选择"我的家庭"作为所属场景，否则角色将无法在场景中显示。

3. **数据验证**：创建完成后，建议在管理后台检查：
   - 场景是否正确创建
   - 角色是否正确关联到场景
   - 图片资源是否正确上传和关联

4. **排序**：角色的排序值已设置，可以在管理后台中调整。

## 后续操作

创建完成后，这些数据将：
- 在管理后台的"场景管理"中显示"我的家庭"场景
- 在"角色管理"中显示8个家庭角色
- 在用户端可以选择"我的家庭"场景进行对话
