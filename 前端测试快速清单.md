# 心域(HeartSphere) - 前端测试快速清单

## 🚀 快速开始测试

本文档提供快速测试清单，帮助测试人员快速执行前端功能测试。

---

## ✅ 一、准备工作（5分钟）

### 环境准备
- [ ] 启动后端服务（端口8081）
- [ ] 启动前端开发服务器（`npm run dev`）
- [ ] 打开浏览器（推荐Chrome）
- [ ] 打开开发者工具（F12）

### 测试账号准备
- [ ] 准备测试用户账号（用户名/密码）
- [ ] 准备管理员账号（如果需要测试管理后台）
- [ ] 准备有效邀请码（如果系统要求）

---

## 🎯 二、核心功能快速测试（30分钟）

### 2.1 用户认证（5分钟）

**快速测试步骤**:
1. [ ] 打开应用 → 检查初始页面显示
2. [ ] 点击"登录" → 输入正确账号密码 → 登录成功
3. [ ] 点击"注册" → 填写信息 → 注册成功
4. [ ] 测试错误密码 → 显示错误提示
5. [ ] 测试空字段 → 显示验证提示

**关键检查点**:
- ✅ 登录后跳转到正确页面
- ✅ Token保存到localStorage
- ✅ 错误提示友好清晰

---

### 2.2 世界管理（5分钟）

**快速测试步骤**:
1. [ ] 进入"真实世界"页面
2. [ ] 点击"创建世界" → 填写信息 → 保存
3. [ ] 检查世界是否显示在列表中
4. [ ] 点击编辑 → 修改信息 → 保存
5. [ ] 点击删除 → 确认删除 → 检查是否移除

**关键检查点**:
- ✅ 创建成功并显示
- ✅ 编辑保存成功
- ✅ 删除功能正常

---

### 2.3 场景管理（10分钟）

**快速测试步骤**:
1. [ ] 点击"创建场景"
2. [ ] 填写场景名称和描述
3. [ ] 上传场景图片（本地文件）
4. [ ] 保存场景
5. [ ] 检查场景是否显示
6. [ ] 测试编辑场景
7. [ ] 测试删除场景

**关键检查点**:
- ✅ 图片上传成功
- ✅ 场景信息完整
- ✅ 所有操作正常

---

### 2.4 角色管理（10分钟）

**快速测试步骤**:
1. [ ] 进入场景 → 点击"创建角色"
2. [ ] 填写基础信息（姓名、年龄、性别等）
3. [ ] 上传角色头像
4. [ ] 填写完整信息（系统指令、MBTI等）
5. [ ] 保存角色
6. [ ] 检查角色是否显示
7. [ ] 测试编辑角色
8. [ ] 测试删除角色

**关键检查点**:
- ✅ 所有字段可以填写
- ✅ 头像上传成功
- ✅ 信息保存完整

---

## 💬 三、对话功能测试（15分钟）

### 3.1 基础对话

**快速测试步骤**:
1. [ ] 选择角色 → 进入对话页面
2. [ ] 发送一条消息："你好"
3. [ ] 等待AI回复
4. [ ] 检查回复是否符合角色设定
5. [ ] 发送多条消息
6. [ ] 退出对话 → 重新进入 → 检查历史记录

**关键检查点**:
- ✅ 消息发送成功
- ✅ AI回复正常
- ✅ 历史记录保存

### 3.2 剧本模式

**快速测试步骤**:
1. [ ] 进入场景 → 选择剧本
2. [ ] 点击"开始剧本"
3. [ ] 查看开场场景和选项
4. [ ] 选择一个选项
5. [ ] 检查是否跳转到下一节点
6. [ ] 继续选择选项，完成剧本流程

**关键检查点**:
- ✅ 剧本正常开始
- ✅ 选项可以点击
- ✅ 节点跳转正确

---

6. [ ] 检查日记是否显示在列表中

- ✅ 日记创建成功
- ✅ 标签保存正确
- ✅ 列表显示正常

1. [ ] 打开设置模态框

3. [ ] 修改AI模型配置（API Key）
4. [ ] 切换对话风格
5. [ ] 保存设置
6. [ ] 验证设置是否生效

**关键检查点**:
- ✅ 所有设置可以修改
- ✅ 设置保存成功
- ✅ 设置立即生效

### 4.3 会员功能（5分钟）

**快速测试步骤**:

3. [ ] 检查删除的项目是否显示
4. [ ] 点击恢复 → 检查是否恢复

## 🐛 五、错误处理快速测试（10分钟）

### 5.1 网络错误

**快速测试步骤**:
1. [ ] 打开开发者工具 → Network标签
2. [ ] 设置为"Offline"模式
3. [ ] 尝试执行需要网络的操作（如创建世界）
4. [ ] 检查错误提示
5. [ ] 恢复网络 → 检查是否自动重试

**关键检查点**:
- ✅ 显示网络错误提示
- ✅ 提供重试选项
- ✅ 数据不丢失

### 5.2 表单验证

**快速测试步骤**:
1. [ ] 尝试提交空表单
2. [ ] 输入无效数据（如负数年龄）
3. [ ] 输入超长文本
4. [ ] 检查验证提示

**关键检查点**:
- ✅ 验证错误提示清晰
- ✅ 不允许提交无效数据

### 5.3 API错误

**快速测试步骤**:
1. [ ] 使用无效Token（修改localStorage中的token）
2. [ ] 执行需要认证的操作
3. [ ] 检查是否跳转到登录页
4. [ ] 测试API配额不足的情况（如果有）

**关键检查点**:
- ✅ 401错误处理正确
- ✅ 自动跳转登录
- ✅ 错误提示友好
