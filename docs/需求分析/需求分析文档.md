**文档版本**: V1.0  
**编写日期**: 2025-12-22  
**项目名称**: 数字生命体交互系统（HeartSphere）  
**英文名称**: Digital Lifeform Interaction System

---

## 一、项目概述

### 1.1 项目背景

数字生命体交互系统（心域）是一个基于人工智能技术的数字生命体交互平台，旨在构建一个与现实世界平行的精神与数据空间。系统通过先进的大语言模型技术，实现了用户与数字生命体（E-Soul）的深度情感交互，支持多时代切片（Era Shard）的访问与管理，为用户提供沉浸式的虚拟角色互动体验。

### 1.2 项目定位

- **产品类型**: 人工智能交互平台 / 数字生命体社交系统
- **目标用户**: 对虚拟角色互动、情感陪伴、创意内容创作感兴趣的用户
- **核心价值**: 提供个性化的数字生命体交互体验，支持多场景、多角色的沉浸式互动

### 1.3 技术架构

- **前端**: React + TypeScript + Tailwind CSS
- **后端**: Spring Boot + JPA + MySQL
- **AI服务**: Spring AI Alibaba + 多模型支持（Gemini、OpenAI、通义千问、豆包等）
- **认证**: JWT + Spring Security
- **部署**: 支持Docker容器化部署

## 二、用户角色与权限

### 2.1 用户角色

#### 2.1.1 普通用户
- 注册、登录、个人信息管理
- 创建和管理数字生命体（角色）
- 创建和管理虚拟世界与时代切片
- 与数字生命体进行智能对话
- 创建和管理日记
- 创建和管理剧本
- 使用会员功能（如已订阅）
- 同步笔记（Notion、Evernote等）

#### 2.1.2 管理员
- 所有普通用户功能
- 系统配置管理
- 用户管理（查看、禁用、删除）
- 邀请码管理
- 系统资源管理
- 订阅计划管理
- 系统数据统计

#### 2.1.3 游客用户
- 浏览系统预置场景和角色
- 查看公开内容
- 需要注册后才能进行交互

### 2.2 权限控制

- **基于JWT的访问控制**: 所有API请求需要携带有效的JWT Token
- **资源隔离**: 用户只能访问自己创建的资源
- **角色权限**: 管理员拥有额外的管理权限
- **邀请码机制**: 可配置是否需要邀请码才能注册

## 三、功能需求分析

### 3.1 用户认证与管理模块

#### 3.1.1 用户注册
**功能描述**: 新用户可以通过多种方式注册账号

**详细需求**:
- 支持用户名+密码+邮箱注册
- 支持微信OAuth2.0授权登录
- 邮箱验证码验证（可选）
- 密码强度验证（至少8位，包含字母和数字）
- 用户名唯一性检查
- 邮箱唯一性检查
- 邀请码验证（如系统启用）
- 注册成功后自动初始化用户数据（创建默认世界、场景等）

**输入**:
- 用户名（必填，3-20个字符）
- 密码（必填，至少8位）
- 确认密码（必填，需与密码匹配）
- 邮箱（必填，需符合邮箱格式）
- 昵称（可选）
- 邀请码（如系统启用）

**输出**:
- 注册成功：返回JWT Token和用户信息
- 注册失败：返回具体错误信息

#### 3.1.2 用户登录
**功能描述**: 已注册用户可以通过多种方式登录

**详细需求**:
- 支持用户名+密码登录
- 支持微信授权登录
- 支持记住登录状态
- 首次登录自动初始化用户数据
- Token有效期管理
- 登录失败次数限制（防止暴力破解）

**输入**:
- 用户名/邮箱
- 密码
- 或微信授权码

**输出**:
- 登录成功：返回JWT Token、用户信息、是否首次登录标识
- 登录失败：返回错误信息

#### 3.1.3 用户信息管理
**功能描述**: 用户可以管理个人资料信息

**详细需求**:
- 查看个人信息（昵称、头像、邮箱、注册时间等）
- 修改昵称
- 上传/更换头像
- 修改密码
- 绑定/解绑微信账号
- 查看登录记录
- 数据统计（场景数、角色数、对话轮数等）

**输入**:
- 修改后的信息（昵称、头像文件、新密码等）

**输出**:
- 更新成功/失败信息

#### 3.1.4 邮箱验证
**功能描述**: 通过邮箱验证码进行身份验证

**详细需求**:
- 发送邮箱验证码
- 验证码有效期（如5分钟）
- 验证码使用次数限制（一次有效）
- 发送频率限制（防止滥用）
- 支持注册验证和密码重置验证

**输入**:
- 邮箱地址
- 验证码类型（注册/重置密码）

**输出**:
- 验证码发送成功/失败
- 验证结果

### 3.4 智能对话交互模块

#### 3.4.1 实时对话
**功能描述**: 与数字生命体进行实时智能对话

**详细需求**:
- 基于大语言模型的实时对话

**输入**:
- 用户消息文本
- 可选图片
- 角色ID
- 场景ID

**输出**:
- 角色回复文本（流式或完整）
- 可选图片生成

#### 3.4.2 对话历史管理
**功能描述**: 管理对话历史记录

**详细需求**:
- 保存对话历史
- 查看历史对话
- 删除对话记录

**输入**:

**输出**:
- 对话历史列表

#### 3.4.3 多模态支持
**功能描述**: 支持文字、图片、语音、视频等多种交互方式

**详细需求**:
- 文字对话（已实现）
- 图片输入和生成（部分实现）
- 语音输入和合成（规划中）
- 视频生成（规划中）

**输入**:
- 多种格式的输入（文本、图片、音频等）

**输出**:
- 多种格式的输出（文本、图片、音频、视频等）

## 四、非功能需求

### 4.1 性能需求

- **响应时间**: 
  - API响应时间 < 500ms（非AI接口）
  - AI对话响应时间 < 5s（首次响应）
  - 页面加载时间 < 2s
  
- **并发能力**: 
  - 支持至少1000并发用户
  - AI服务支持至少100并发请求
  
- **吞吐量**: 
  - API吞吐量 > 1000 req/s
  - 数据库查询优化，支持复杂查询

### 4.2 可用性需求

- **系统可用性**: 99.5%以上
- **故障恢复**: 支持自动故障恢复
- **数据备份**: 定期数据备份，支持数据恢复

### 4.3 安全性需求

- **认证安全**: 
  - JWT Token加密
  - 密码BCrypt加密存储
  - Token过期机制
  
- **数据安全**: 
  - 用户数据隔离
  - SQL注入防护
  - XSS攻击防护
  - CSRF防护
  
- **API安全**: 
  - 接口权限验证
  - 请求频率限制
  - 敏感数据加密传输

### 4.4 可扩展性需求

- **水平扩展**: 支持多实例部署
- **模块化设计**: 易于添加新功能模块
- **插件机制**: 支持第三方插件集成

### 4.5 兼容性需求

- **浏览器兼容**: 支持Chrome、Firefox、Safari、Edge等主流浏览器
- **移动端兼容**: 支持iOS和Android
- **API兼容**: 保持API向后兼容

### 4.6 可维护性需求

- **代码规范**: 遵循代码规范和最佳实践
- **文档完善**: 提供完整的开发文档和API文档
- **日志系统**: 完善的日志记录和监控
